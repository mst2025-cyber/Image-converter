画像タイプ別の目安

出力形式はバッチ全体で共通です。アップロードしたすべての画像が同じ形式で変換されます。

画像タイプ 向く出力 品質の目安 補足
写真 JPG / WebP 75〜85% サイズと画質のバランスが取りやすい
イラスト WebP / JPG 85〜95% 境界・グラデーションは高め推奨
図形・表 WebP / JPG 90〜100% 線や文字のにじみを抑える。元が小さい PNG なら変換不要なことも
スクリーンショット WebP 推奨 90〜100% 文字の読みやすさ優先

品質と圧縮率について

品質スライダーと圧縮率は比例しません。

  • 品質 … 変換時にどれだけ細部を捨てるかの設定
  • 圧縮率 … 元ファイルサイズに対して実際にどれだけ小さくなったか（結果テーブルに表示）

画像の中身によって結びつき方が変わります。

  • 図形・表 … 高品質でも圧縮率が高い（よく小さくなる）ことが多い
  • すでに小さい PNG … 圧縮率 0% やマイナスもあり得る
  • 写真 … 品質を下げるほどファイルは小さくなりやすいが、完全な直線ではない
  • JPEG の再変換 … 圧縮率はあまり動かないのに、画質だけ劣化することがある

不可逆変換の注意

  • JPG と WebP（本アプリ）はいずれも不可逆（ロッシー）圧縮です
  • 同じ画像を繰り返し変換するたびに画質が劣化します（品質 100% でも可逆ではありません）
  • 何回変換されたかの履歴はファイルに残りません
  • 元が JPEG の画像を再度 JPG に変換する用途には向きません

WebP について

本アプリの WebP 出力はブラウザの Canvas API を利用した不可逆 WebP のみです。可逆（ロッスレス）WebP には対応していません。
